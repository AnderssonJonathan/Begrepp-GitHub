Github flow

I ett projekt för att t.ex. koda en hemsida så kan GitHub flow följas. Det innebära att de som kodar kommer att arbeta utifrån en mall med särskilda steg som skall följas löpande under projektets gång. Då GitHub flow följs kommer projekt att sparas kontinuerligt på ens GitHub konto. På så sätt kommer det finnas en struktur genom hela projektet och det blir enkelt att gå tillbaka och granska för att se ändringar antingen under själva processen eller efteråt. GitHub är en molnbaserad tjänst och ens arbete blir på så sätt tillgängligt online och smidigt att dela med andra.

Att följa GitHub flow är fullt möjligt att göra enbart via GitHubs egna sida men det är mycket vanligt att arbeta med sitt projekt på ens egna dator. För att göra detta krävs Git och stegen för att installera detta finns på GitHubs hemsida men kommer inte att tas upp i denna guide. Men jag kommer att i de olika stegen att också inkludera hur du kan arbeta med din kod lokalt. Till det använder jag Git, VS-code och terminalen i det.

Stegen för GitHub flow:

- Skapa ett repo

Det första att börja med är att skapa ett repository (repo). Detta är som en mapp där alla filer till ens projekt ska placeras. Ens repo måste ha ett namn och därefter finns det olika inställningar som är möjliga att lägga till eller ändra. T.ex. kan du välja ifall det ska vara public (öppet för alla) eller privat (endast tillgängligt för dig). Det finns en möjlighet att skapa repot med en README-fil vilket kan vara användbart då det behöver finnas något i repot för att alla funktioner ska vara tillgängliga.

När ditt repo ligger uppe är det sedan möjligt att kopiera länken till repot. Det är vanligt att också ha en lokal repo-mapp på sin egen dator. Med hjälp av länken från GitHub och Git kan man skapa ett lokalt repo till sin repo-mapp. Den är då en klon av hur repot på GitHub såg ut när länken till det repot kopierades. Det är sedan möjligt att öppna detta lokala repo på sin dator med ett kodeditorprogram som t.ex. VS-code.

- Ändringar - commits & push

I sin nya bransch är det sedan möjligt att spara sitt arbete till den nuvarande branschen viket görs med commits. Varje commit behöver en kommentar där man kortfattat förklarat vad man gjort t.ex. “Created logo.”. Därefter är commiten redo att pushas fortfarande till den nya branschen. Med hjälp av detta kan laddas projektet upp i molnet och andra deltagare i projektet kan gå in och titta på det du arbetar med och kanske har du frågor till dem där.

I VS-code behöver ändringar läggas till med hjälp av ett kommando i terminalen t.ex. “git add .” som därmed registrerar alla ändringar. Därefter görs en commit med följande kommando “git commit –m “dinKommentar”. Sist görs en push med kommandot “git push –u origin namnPåDinBranch”.

Skapa pull request - Merge pull request

Nu är det möjligt att skapa en pull request för sin branch vilket innebär att du frågar dina kollegor i projektet om feedback på dina ändringar. Här kan inställningar ha gjorts för projektet att t.ex. x-antal kollegor måste godkänna en pull request. Om allt ser bra ut så kan de som granskar välja merge vilket då kommer innebära att som finns i denna bransch kommer att läggas till main-branschen.

Radera din branch

Slutligen när ens branch har merge:at till main-branschen så är det möjligt att radera denna bransch eftersom den inte används längre. En vanlig policy inom projekt med GitHub flow är att det är skaparen av branschen som också raderar den.

Flexbox

Precis som namnet Flexbox indikerar så handlar det om en flexibel box. Flexbox Layout Module är dess fullständiga namn är och det är alltså en komplett modul som innehåller en hel uppsättning av egenskaper. Flexbox möjliggör det för oss att mer effektivt skapa en dynmaisk t.ex. hemsida som kan ändra storleken på element och dess innehåll utifrån om sidan besöks från en mobil, surfplatta eller dator. Vissa av egenskaperna kan appliceras på containers alltså parent element och då används flex container medans flex items används på children.

DOM (Document Object Model)

DOM är ett API som används för att bygga hemsidor. Det är mest vanligt förekommande att det används genom JavaScript men det är byggt för att kunna användas till vilket programspråk som helst. För objekt som utgör strukturen och innehållet för exempelvis en hemsida är DOM datarepresentationen av dessa. Vilket gör så att program som t.ex. JavaScript kan ändra strukturen av dokumentet som utgör själva sidan. Det som möjliggör detta är att DOM representerar innehållet på sidan i form av noder och objekt på så sätt kan programspråken interagera med innehållet på sidan. Fin kod

Vid programmering handlar det inte bara om att du själv ska kunna förstå din kod. Som programmerare är det vanligt att sträva efter en kod som är enkel att förstå det vill säga en “fin kod”. Att det finns en logisk uppbyggnad, att klasser, id osv. har namn som tydligt indikerar vad de syftar till, att koden följer samma struktur, det kanske finns nödvändiga radbrytningar som gör koden tydligare osv.

Ramverk

Att bygga upp en t.ex. komplett hemsida görs idag sällan från grunden. Ramverk möjliggör det för oss att bygga en hemsida på en redan färdig grund vilken det är möjligt för oss att hämta de byggstenar som vi vill använda oss av. Det finns idag massvis med olika ramverk, de lämpas sig till olika typer av uppgifter och är oftast kopplade till ett särskilt programspråk. Exempel på ramverk:

- AngularJS

Ett av de vanligaste web-ramverken är AngularJS. Det är ett front-end JavaScript ramverk som är utvecklat och stöds av Google. AngularJS innehåller funktioner som t.ex. tvåvägsdatabindning vilket minskar utvecklingstiden. Olika kodbitar kan också lättare interagera med varandra med hjälp av s.k. beroendeinjektioner i AngularJS.

- Express

Till Node.js finns Express som är ett back-end ramverk. Det gör det möjligt att börja konfigurera servrar med väldigt lite dataresurser. Det är skrivet i JavaScript, är gratis och open-source och det har också ett rykte av att vara väldigt flexibelt. Enligt Express själva så tillhandahåller det ett tunt lager av grundläggane webbapplikationer utan att skymma Node.js-funktioner.
